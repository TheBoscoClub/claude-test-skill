Context Retention Architecture

Brain-inspired redesign: distributed recovery with graduated pressure response

Three-Source Recovery Model

No single file is the recovery mechanism. Three independent sources combine to reconstruct full working state after compaction or crash.

Source 1 — Intent
Wallet Card
.claude-task-state.md
~30 lines, what & why
Source 2 — Actual Changes
Git State
git diff + git log
ground truth of modifications
Source 3 — History
Session Record
SESSION_RECORD_*.md
audit trail, not recovery
↓
Full Context Reconstructed
Current task + files touched + next step + actual diffs

Distributed Memory

Brain: hippocampus + cortex + cerebellum
System: wallet card + git + session record

Consolidate During Idle

Brain: sleep consolidation, not constant journaling
System: write at milestones, not every tool call

Emergency Card

Brain: medical ID bracelet (name, meds, contacts)
System: wallet card (~30 lines of structured state)

Quiet Under Pressure

Brain: goes quiet during stroke, not louder
System: context monitor goes silent at Critical (>7 MB)

Graduated Context Pressure Response

Thresholds calibrated from actual compaction-sizes.log data. Compaction fires at 1.4–12 MB (median ~6–7 MB). Old thresholds (600 KB / 800 KB) were 5–10x too low.

Normal
< 3 MB
Elevated
3–5 MB
High
5–7 MB
Critical
> 7 MB
0 MB 3 MB 5 MB 7 MB → compaction

What Was Eliminated

Old Behavior Problem New Behavior
additionalContext injection Feedback loop: hook tells Claude to write, write triggers hook Never used at any level
Session record writes from hook 5,000+ line records filled with duplicate monitor entries Hook never touches session record
No cooldown at CRITICAL Fires every tool call, spamming the dying context window Goes completely silent at Critical
600 KB / 800 KB thresholds Triggered panic at ~10% of actual compaction distance 3 / 5 / 7 MB based on real calibration data

Old System vs New System

Aspect Old System New System
Recovery target Session record (monolithic, 5,000+ lines) Wallet card (~30 lines) + git state
Write cadence Every tool call / every ~3 minutes Milestones only (commits, errors, decisions)
Context monitor Writes to session record, injects additionalContext, no cooldown Observe-only, one-time actions, silent at Critical
Feedback loops Hook → Claude writes → hook fires again → infinite Impossible — no additionalContext, no session record writes
Session record role Recovery mechanism + audit trail + monitor dump Audit trail only (milestone entries)
Under pressure Gets louder (more writes, more injection) Goes quiet (trusts PreCompact hook)
Recovery time Read 5,000+ line file, parse through noise ~30 seconds: wallet card + git diff + git log
Concurrency flock-based locking for session record writes No locking — only master session writes wallet card
Subagent writes Write to session record continuously Append summary on completion only
Thresholds 600 KB / 800 KB (guessed) 3 / 5 / 7 MB (calibrated from compaction-sizes.log)
